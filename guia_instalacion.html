Antes de empezar
Archivos base que necesitas

Estos son los archivos maestros — los genéricos que sirven para cualquier cliente. Los tienes en tu Drive/computadora y los reutilizas para cada instalación.

ArchivoDescripcionQuien lo modifica
ScriptBase_v1.jsApps Script principal del sistemaClaude
aceptar_v2.htmlPortal del cliente para firmarClaude
pago.htmlPágina de pago con datos bancariosClaude
terminos.htmlTérminos y condicionesClaude
InstaladorMaestro.jsCrea Sheets, Form, Drive automáticamenteTú
intake_v4.htmlFormulario de intake (ya subido, no cambia)Fijo
💡
Claude modifica los primeros 4 archivos con los datos del cliente y te los devuelve listos. El InstaladorMaestro lo ejecutas tú desde la cuenta del cliente. El intake nunca cambia.
1
Paso 1
Recibir el correo de intake
⏱ 0 min — automático

El cliente llena el intake_v4.html y el sistema te manda automáticamente un correo a inmueblesaudiovisuales@gmail.com con todo lo que necesitas.

1
El cliente llena el formulario
Le mandas el link tudominio.com/intake_v4.html por WhatsApp o correo. Tarda 5-10 minutos en llenarlo.
2
Te llega el correo
Revisas tu correo. El asunto dice Nuevo cliente — [Nombre del negocio]. Contiene tres secciones importantes:
📋 Resumen del clienteTodos los datos del negocio, banco, servicio, etc.
🤖 Prompt maestroPara pegar en Claude junto con los 4 archivos
📝 Prompt Google FormPara crear el Form con Gemini
3
Copia el prompt maestro
Copia todo el texto del bloque "Prompt maestro" del correo. Lo usarás en el siguiente paso.
⚠️
Si no llega el correo, ve a script.google.com → tu proyecto ReceptorIntake → Ejecuciones y revisa si hay errores. Probablemente necesitas redesployar con nueva versión.
2
Paso 2
Claude genera los archivos del cliente
⏱ 3 min
1
Abre un chat nuevo en claude.ai
Importante: un chat nuevo, no el que usas normalmente. Así Claude tiene contexto limpio.
2
Sube los 4 archivos base
Arrastra o selecciona estos archivos desde tu computadora:
ScriptBase_v1.js
El script genérico sin datos de ningún cliente
aceptar_v2.html
El portal genérico del cliente
pago.html
La página de pago genérica
terminos.html
Los términos genéricos
3
Pega el prompt maestro
Pega el texto completo que copiaste del correo (el bloque "Prompt maestro") y envía. Claude leerá los archivos, llenará los CONFIGs y generará los archivos modificados + el contrato en Word.
✓
Claude tardará 30-60 segundos. Al terminar te entregará los 4 archivos modificados y el Contrato_Template.docx.
4
Descarga los 5 archivos que te da Claude
Guárdalos en una carpeta con el nombre del cliente. Ej: Clientes/Proposal Inc/
3
Paso 3
Subir los HTMLs a Cloudflare Pages
⏱ 2 min
1
Ve a dash.cloudflare.com
Inicia sesión con tu cuenta de Cloudflare (la de inmueblesaudiovisuales).
2
Workers & Pages → Pages → Create a project
Selecciona "Upload assets" (no conectes GitHub).
dash.cloudflare.com/pages/new
Nombre del proyecto
proposalinc
Usa el slug del negocio, en minúsculas sin espacios
📁 Arrastra aquí los 3 archivos HTML
aceptar_v2.html · pago.html · terminos.html
3
Sube los 3 archivos HTML
Solo los HTMLs — no el .js ni el .docx. Arrastra los 3 a la vez.
⚠️
Asegúrate de subir aceptar_v2.html, no el aceptar.html antiguo.
4
Deploy y copiar la URL
Cloudflare te da una URL permanente. Anótala — la necesitas en los siguientes pasos.
URL del proyectoproposalinc.pages.dev
URL_ACEPTARproposalinc.pages.dev/aceptar_v2.html
URL_PAGOproposalinc.pages.dev/pago.html
5
Verifica que los archivos cargan
Abre proposalinc.pages.dev/aceptar_v2.html en el navegador. Debe cargar la página (aunque diga "Token requerido" — eso es normal).
4
Paso 4
Ejecutar el InstaladorMaestro
⏱ 2 min

Este script crea automáticamente el Google Sheets, el Google Form, la carpeta en Drive y copia el template del contrato. Todo en la cuenta del cliente.

ℹ️
Requisito: Debes estar loggeado en la cuenta de Gmail del cliente cuando ejecutes este script. Pídele al cliente acceso o entra desde su computadora.
1
Ve a script.google.com desde la cuenta del cliente
Asegúrate de que el Gmail en la esquina superior derecha es el del cliente, no el tuyo.
2
Nuevo proyecto → pegar InstaladorMaestro.js
Crea un nuevo proyecto de Apps Script, borra el código default y pega el contenido completo de InstaladorMaestro.js.
3
Llena el bloque DATOS_CLIENTE
Al inicio del script hay un bloque con los datos del cliente. Llénalo con la información del correo de intake:
JavaScriptInstaladorMaestro.js
const DATOS_CLIENTE = { NOMBRE_NEGOCIO : 'Proposal Inc', EMAIL_FOTOGRAFO : 'proposalincmx@gmail.com', WHATSAPP : '528112345678', COLOR_ACENTO : '#1A1A2E', TEMA : 'claro', BANCO : 'BBVA', CLABE : '111111111111111111', // ... resto de datos del correo de intake TEMPLATE_DOC_ID : 'PEGA_TU_ID_DEL_DOC_TEMPLATE', };
⚠️
TEMPLATE_DOC_ID es el ID de tu Google Doc maestro del contrato (el que tienes en tu propio Drive). Debe estar compartido como "Cualquiera con el enlace puede ver".
4
Ejecuta autorizar()
Selecciona autorizar en el menú de funciones → clic en "Ejecutar". Google te pedirá permisos — acepta todo.
5
Ejecuta instalar()
Selecciona instalar → "Ejecutar". Espera ~20-30 segundos. Verás logs en la consola.
Log esperado✓ Carpeta creada: 1abc...
✓ Sheets creado: 1def...
✓ Form creado y vinculado: 1ghi...
✅ INSTALACIÓN COMPLETA
6
Abre el Sheets que se creó y anota los IDs
El instalador crea una hoja llamada "Instalación" con todos los IDs. Necesitarás:
SHEETS_ID1wHlh... (lo que dice en la hoja)
FOLDER_ID1zyTh... (lo que dice en la hoja)
TEMPLATE_CONTRATO_ID1f9E-... (lo que dice en la hoja)
URL del Formdocs.google.com/forms/... (para dársela al cliente)
5
Paso 5
Pegar el script y hacer el deploy
⏱ 3 min
1
Abre el Sheets del cliente → Extensiones → Apps Script
Desde el Sheets que creó el instalador. Esto vincula el script al Sheets automáticamente.
2
Borra el código default y pega el ScriptBase_v1.js del cliente
El archivo que te generó Claude (no el genérico). Ya tiene el CONFIG con los datos del cliente.
⚠️
Verifica que el CONFIG del script tiene SHEETS_ID, FOLDER_ID y TEMPLATE_CONTRATO_ID con los IDs reales de la hoja "Instalación". Si dicen "PENDIENTE", llénalos ahora.
3
Guarda el script (Ctrl+S o Cmd+S)
Dale un nombre al proyecto, por ejemplo Contratos — Proposal Inc.
4
Implementar → Nueva implementación
Configura exactamente así:
TipoAplicación web
Ejecutar comoYo (la cuenta del cliente)
Quién tiene accesoCualquier usuario
🚨
"Cualquier usuario" es CRÍTICO. Si dice "Solo yo" o "Usuarios de Google Workspace", el sistema no funcionará para los clientes.
5
Copia la URL del deploy
Se ve así:
https://script.google.com/macros/s/AKfycbwi...largo.../exec
Guárdala — es tu SCRIPT_URL. La necesitas en el siguiente paso.
6
Paso 6
Llenar los IDs pendientes
⏱ 2 min

Ahora que tienes la SCRIPT_URL del deploy y las URLs de Cloudflare, completa los últimos valores pendientes.

1
En el ScriptBase_v1.js — actualiza el CONFIG
Busca las líneas PENDIENTE y reemplázalas:
Antes
URL_ACEPTAR : 'PENDIENTE', URL_PAGO : 'PENDIENTE',
Despues
URL_ACEPTAR : 'https://proposalinc.pages.dev/aceptar_v2.html', URL_PAGO : 'https://proposalinc.pages.dev/pago.html',
2
En aceptar_v2.html y pago.html — actualiza SCRIPT_URL
Abre cada archivo en un editor de texto y busca SCRIPT_URL : 'PENDIENTE':
aceptar_v2.html y pago.html
SCRIPT_URL : 'https://script.google.com/macros/s/AKfycbwi.../exec',
3
Redesploya el script con nueva versión
En Apps Script: Implementar → Gestionar implementaciones → ✏️ editar → Nueva versión → Implementar. La URL no cambia.
4
Re-sube los HTMLs actualizados a Cloudflare
En Cloudflare Pages → tu proyecto → "Upload assets" → sube los 3 HTMLs nuevamente. El deploy es automático.
7
Paso 7
Autorizar permisos e instalar triggers
⏱ 1 min
1
En Apps Script del cliente → ejecuta autorizar()
Selecciona la función autorizar en el menú y dale "Ejecutar". Google pedirá permisos para Gmail, Drive, Sheets y Docs. Acepta todo.
⚠️
Puede aparecer una pantalla de advertencia "Esta app no está verificada". Clic en "Configuración avanzada" → "Ir a [nombre del proyecto] (no seguro)". Es normal para scripts personales.
2
Ejecuta instalarTodos()
Selecciona instalarTodos y dale "Ejecutar". Esto instala dos triggers automáticamente:
onFormularioSubmit✓ Se dispara cuando el fotógrafo llena el Form
limpiarTokensViejos✓ Limpia tokens expirados cada domingo
✓
Puedes verificar en el menú izquierdo de Apps Script → ícono de reloj (Triggers) que aparecen los dos triggers instalados.
8
Paso 8 — El mas importante
Prueba de punta a punta
⏱ 3 min
ℹ️
Usa tu propio correo como cliente de prueba. Así puedes verificar que todos los correos llegan correctamente.
1
Abre el Google Form y llénalo con datos de prueba
Usa datos reales para que el PDF quede bien formado:
Nombre de clienteAna Martinez (prueba)
Paquete ElegidoPaquete Básico
Precio5000
Fecha de grabacióncualquier fecha futura
Entregables30 fotos editadas · 1 reel de 60 seg
2
Verifica el correo al fotógrafo
Debe llegar al Gmail del cliente en menos de 1 minuto con el link de firma. Si no llega en 2 minutos, ve a Apps Script → Ejecuciones y revisa el error.
3
Abre el link de firma y llena el formulario del cliente
Verifica que el portal carga bien, con los colores correctos, el nombre del negocio y las preguntas configuradas. Usa tu propio correo como el del cliente.
4
Verifica los correos al cliente
Revisa TU correo (el que pusiste como cliente de prueba). Deben llegar:
  • Correo con el PDF del contrato adjunto — abre el PDF y verifica que los datos estén bien
  • Datos bancarios para transferencia — CLABE, banco, titular correctos
  • Botón "Ver opciones de pago" — lleva a pago.html con los datos correctos
5
Verifica el correo al fotógrafo con los botones
El Gmail del cliente debe tener un correo de "Contrato firmado" con:
  • Resumen completo del cliente — nombre, correo, teléfono, dirección, etc.
  • Botón "Agregar al calendario" — debe abrir Google Calendar con el evento pre-llenado
  • Botón "Confirmar pago recibido" — debe funcionar cuando se da clic
6
Da clic en "Confirmar pago recibido" y verifica el correo de confirmación
Tu correo de prueba debe recibir el correo final de confirmación: "Tu sesión está confirmada". Si llega, el sistema funciona completo.
🎉
Si todos los correos llegaron y el PDF se generó correctamente — el sistema está listo para entregar al cliente.

Referencia
Errores comunes y soluciones
No llega el correo al fotógrafo
1. Revisa Apps Script → Ejecuciones
2. Si no aparece ninguna: el trigger no está instalado — ejecuta instalarTodos()
3. Si aparece con error: lee el log y busca el mensaje
Error: "Token inválido" al abrir el link
La SCRIPT_URL en aceptar_v2.html es incorrecta o PENDIENTE.
Verifica que apunta al deploy correcto de la cuenta del cliente, no al tuyo.
El PDF se genera vacío o con errores
Verifica que el TEMPLATE_CONTRATO_ID es correcto y que el Doc está compartido como "Cualquiera con el enlace puede ver".
401 Unauthorized al enviar el formulario
El deploy está configurado con acceso limitado.
Reimplementa: Gestionar implementaciones → editar → cambiar acceso a "Cualquier usuario".
Error: "Hoja de contratos no encontrada"
El SHEETS_ID en el CONFIG apunta al Sheets incorrecto.
Copia el ID de la URL del Sheets del cliente.
El portal del cliente no carga (404)
Los archivos no se subieron correctamente a Cloudflare.
Verifica que se llame aceptar_v2.html y no aceptar.html.
💡
Regla de oro: cuando algo falla, siempre ve primero a Apps Script → Ejecuciones. El log dice exactamente qué salió mal y en qué línea.
✓
Antes de entregar al cliente
Checklist de entrega
  • El Google Form tiene los 5 campos exactos con los nombres correctos (mayúsculas, acentos)
  • El Sheets está vinculado al Form y tiene las hojas "Contratos" y "Tokens"
  • El CONFIG del script no tiene ningún "PENDIENTE" — todos los IDs y URLs están llenados
  • MODO_BORRADOR está en false en el script
  • Los dos triggers están instalados (verifica en el menú de reloj)
  • La Web App está deployada como "Cualquier usuario"
  • Los 3 HTMLs en Cloudflare tienen SCRIPT_URL correcta (no PENDIENTE)
  • El PDF del contrato de prueba se generó correctamente con todos los datos
  • Todos los correos de prueba llegaron (fotógrafo + cliente + confirmación)
  • El botón "Agregar al calendario" funciona
  • El botón "Confirmar pago recibido" funciona
  • La página pago.html muestra los datos bancarios correctos
  • El cliente sabe cómo usar el Form — explícale en 2 minutos cómo funciona
Sistema listo

Si todos los puntos del checklist están completos, el sistema está funcionando. El cliente solo necesita saber cómo llenar el Form — el resto es automático.
